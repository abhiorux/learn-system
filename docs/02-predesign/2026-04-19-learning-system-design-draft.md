# 学习系统设计讨论记录

日期：2026-04-19

## 需求背景

用户有一个图形渲染学习仓库（OpenGL + Vulkan），包含按课程组织的源码和文档。用户希望设计一个独立的学习管理系统，具有可复用性，适用于其他学习项目。

---

## 核心需求

1. **知识管理**：建立知识库，支持知识入库、更新、关联、检索
2. **学习应用**：根据知识库自动推荐学习内容
3. **学习反馈**：用户反馈影响后续推荐
4. **可复用性**：其他学习项目也能使用同一套系统

---

## 设计原则

- 配置驱动，而非硬编码
- 本地计算为主，保护隐私
- 轻量化反馈收集，不打断学习心流
- 多项目通过配置隔离

---

## 数据模型

### 核心实体

| 实体 | 作用 |
|------|------|
| Knowledge Node | 知识节点（知识点、笔记、踩坑记录） |
| Learning Record | 用户的学习记录（时间、时长、理解度） |
| Feedback Event | 反馈事件（显式/隐式） |
| Learning Path | 学习路径（有向无环图） |

### 关键关系

```
知识库 → 知识图谱（依赖关系）
用户反馈 → 推荐引擎 → 学习路径 → 前端展示
遗忘曲线 → 复习提醒
```

---

## 推荐算法思路

### 核心原则：平衡「探索」与「巩固」

### 推荐得分公式

```
推荐得分 = f(知识重要性, 用户掌握度, 遗忘曲线, 学习目标)
```

### 策略规则

| 情况 | 策略 |
|------|------|
| 前置知识未掌握 | 强制回退 |
| 多次"没理解" | 降低难度或换资源 |
| 标记"踩坑" | 推荐踩坑记录 + 练习 |
| 3天未复习 | 按艾宾浩斯安排 |
| 连续完成多个 | 适当加速 |
| 跳过某知识点 | 分析原因 |

### 遗忘曲线模型

```
掌握度(t) = 初始掌握度 × e^(-t/τ)
τ 由用户表现个性化调整
```


---

## 后端职责

- **知识管理**：入库、关联、更新、检索
- **学习分析**：用户画像、路径规划、效果评估、异常检测
- **反馈处理**：显式反馈、隐式反馈、反馈加权

## 前端职责

- **学习界面**：任务展示、笔记编辑、反馈入口
- **进度视图**：图谱可视化、日历热力图、薄弱点列表
- **个性化**：主题、推荐强度、每日目标
